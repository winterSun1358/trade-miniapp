## 《微信小程序开发》考查内容及评分标准 

## 一、课程设计目的 

《微信小程序开发》课程设计旨在系统检验学生对微信小程序开发基础知识 的掌握程度与综合运用能力。考核聚焦于小程序框架核心概念、常用组件与 API 的实践应用，要求学生分组完成一个功能较完整的基础项目。通过项目开发过程， 重点考察学生对页面布局、数据绑定、事件处理、本地存储及网络请求等基础环 节的理解与动手能力，锻炼其分析问题、查阅文档与调试代码的技能，为后续深 入学习复杂商业应用开发奠定坚实基础。 二、课程设计要求 

1. 分组要求： 

学生自愿选题，自由组队，每组人员 不超过 **4** 人 ，每组设组长 1 名，负责组 内任务的分工并组织交流与讨论，确保每名组员均有实际编程任务、各司其职， 在规定时间内完成课程设计任务。 

## 2. 界面设计： 

小程序界面应主题明确、结构清晰，至少包含首页、功能页、详情页或个人 中心等 3 个页面；页面之间能够通过 tabBar、navigator 或导航 API 正确跳转。 

合理使用 view、text、image、button、input、form、swiper、scroll-view、picker 等基础组件，采用 Flex 布局完成列表、卡片、表单、导航栏等常见界面效果。 

页面配色、字体、间距、图标与图片风格应保持统一，交互提示明确，表单 输入具备必要校验，能够适配常见手机屏幕，不出现明显错位、遮挡、空白或滚 动异常。 

## 3. 技术要求：

- 2) 至少完成 3 个可运行页面，包含清晰的页面路由和导航逻辑，项目启 动后能够正常浏览主要功能。 

- 3) 能够综合使用数据绑定、条件渲染 wx:if、列表渲染 wx:for、事件绑定 和 setData 完成动态页面展示。
